Installation projet :
composer create-project symfony/skeleton demo

Installation du module server en mode de dev :
composer require server --dev

Lancer l'application dans le serveur :
php bin/console server:run

Créer un controleur :
php bin/console make:controller

Fichier de configuration Symfony :
.env

Variable dans .env pour renseigner les informations de la base de donnée :
DATABASE_URL=mysql://root:@127.0.0.1:3306/blog

Créer une entité dans Symfony :
php bin/console make:entity

Création du fichier de migration :
php bin/console make:migration

Lancement du ou des fichiers de migration pour mettre à jour la base de données :
php bin/console doctrine:migrations:migrate

Installation du module fixtures en mode dev :
composer require orm-fixtures --dev

Création d'une fixture :
php bin/console make:fixtures

Installation projet avec Symfony 4 :
composer create-project "symfony/website-skeleton:^4.0" appli

Lancement application :
php -S localhost:8000 -t public
symfony server:start

Installation serveur pour mode de développement puis lancement application :
composer require server --dev
php bin/console server:run

Visualiser les services et interfaces disponibles dans l'application :
php bin/console debug:autowiring

Création base de données en ligne de commande :
php bin/console doctrine:database:create

Créer une entité :
php bin/console make:entity

Créer fichier de migration pour la base de données :
php bin/console make:migration

Appliquer les informations de migration pour mettre à jour la base de données :
php bin/console doctrine:migrations:migrate

Ajouter champ dans table :
php bin/console make:entity <NOM_ENTITE>
php bin/console make:migration
php bin/console doctrine:migrations:migrate

Ajouter bibliothèque slugify (slug) au projet :
composer require cocur/slugify
